# Design Log

> Append-only record of the design process. **Update this file as the campaign runs** (and as the paper/repo are read more closely). Two parts: a **per-candidate metrics table** (one row per design ŌĆö filtered *and* top-ranking alike) and a **round decisions log** (narrative of what changed between rounds and why). Method/cutoff context lives in [`02_methods_and_inputs.md`](02_methods_and_inputs.md).

## Candidate metrics table

One row per design ŌĆö record every candidate evaluated, not just survivors, so filtered designs document what fails.

**Partial scores are expected.** Designs are scored **after each step**, and most are killed early in the funnel. If a candidate fails at an early step, **leave the downstream columns blank** (`ŌĆö`) ŌĆö there's no need to run the rest. So a filtered row may carry only its early metrics plus a `notes` reason, while shortlisted/AF3-confirmed rows fill in more.

The **full per-design schema lives in a CSV** (`designs/candidates.csv`) ŌĆö the source of truth ŌĆö with these columns:

> `candidate_id, round, status, interaction_hypothesis, structure, length, net_charge, ss_pctH, ss_pctE, ss_pctL, frac_exposed_hydrophobic, polyAla, af2_pae_interaction, af2_monomer_pLDDT, binder_rmsd, af3_iptm, af3_binder_pep_iptm, ipSAE_pep_binder, cms_Asp_p5, hbond_contacts, hydrophobic_contacts, rosetta_ddG, shape_complementarity, packstat, interface_unsat_hbonds, buried_unsat_hbonds, n_Lys_interface, n_Met_interface, delta_pae_specificity, mpnn_spec_score, notes`
>
> Thresholds for reference: `af2_pae_interaction < 5`; `af3_iptm Ōēź 0.90`; `af3_binder_pep_iptm Ōēł 0.77ŌĆō0.91`; `delta_pae_specificity` more negative = more specific; `cms_Asp_p5` higher = better contact on the mutant.
